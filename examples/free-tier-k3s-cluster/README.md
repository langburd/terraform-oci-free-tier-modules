# Free Tier K3s Cluster Example

Deploys a fully functional K3s cluster on OCI Always Free Tier using Terraform and Ansible.

## Architecture

- 1 VCN with 1 public subnet
- 1 Security List with K3s-required rules
- 4x `VM.Standard.A1.Flex` instances (1 OCPU, 6 GB each): 1 server + 3 agents
- K3s deployed via [k3s-ansible](https://github.com/k3s-io/k3s-ansible)

## Free Tier Resource Usage

| Resource | Used | Free Limit |
|----------|------|-----------|
| Arm OCPUs | 4 | 4 |
| Arm Memory | 24 GB | 24 GB |
| Boot Volumes | 200 GB (4 × 50 GB) | 200 GB |
| VCNs | 1 | 2 |

> **Note:** This configuration uses exactly all Arm and storage free tier resources.

## Prerequisites

- OCI CLI configured with a valid profile
- Terraform >= 1.14
- `ansible-playbook` binary installed on the Terraform runner
- `ansible` Python package: `pip3 install ansible`

## Usage

```bash
terraform init
terraform plan
terraform apply
# After K3s is deployed:
$(terraform output -raw kubeconfig_command)
# Then fix the server address:
# kubectl config set-cluster default --server=https://<server_ip>:6443
kubectl get nodes
```

## Notes

- SSH keypair is generated by Terraform and written to `.ssh/k3s-key` (600 permissions)
- Cloud-init installs Python 3 on nodes (required by Ansible)
- A1.Flex instances may show "Out of host capacity" in new accounts; retry or try different ADs
- For mixed-arch (2 Arm + 2 AMD): change 3 agents to 1 Arm + 2 AMD Micro instances
